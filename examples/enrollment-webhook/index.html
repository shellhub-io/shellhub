ShellHub calls this endpoint synchronously at device registration. The returned decision maps to the device's initial status (accept → accepted, reject → rejected, pending → manual queue). ShellHub fails closed to pending on timeout, non-2xx, an unparseable body, or an unknown decision.

Async / deferred: returning defer lands the device pending and keeps the payload's token-authenticated callback_url. Decide later with the Accept / Reject buttons on the row — the server POSTs the decision to that URL (the token in the URL is the only credential) and shows the callback's HTTP outcome.
