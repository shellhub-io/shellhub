# Yocto Project

Add the ShellHub agent to your Yocto-based Linux image using the meta-shellhub layer.

## Prerequisites

- A Yocto Project release supported by the layer (currently `styhead`, `whinlatter`, or `wrynose` — see `LAYERSERIES_COMPAT` in the layer)
- Internet access to fetch the agent source during build

## Setup

### 1. Clone the meta-shellhub layer

```bash
$ git clone https://github.com/shellhub-io/meta-shellhub.git
```

### 2. Add the layer to your build

In your `bblayers.conf`:

```
BBLAYERS += "/path/to/meta-shellhub"
```

### 3. Configure in `local.conf`

```bash
CORE_IMAGE_EXTRA_INSTALL += "packagegroup-shellhub-runtime"
SHELLHUB_TENANT_ID = "YOUR_TENANT_ID"
```

### Optional configuration

```bash
SHELLHUB_SERVER_ADDRESS = "https://cloud.shellhub.io"
SHELLHUB_PREFERRED_HOSTNAME = "my-yocto-device"
SHELLHUB_PRIVATE_KEY = "/etc/shellhub.key"
SHELLHUB_KEEPALIVE_INTERVAL = "30"
```

If `SHELLHUB_SERVER_ADDRESS` is not set, it defaults to `https://cloud.shellhub.io`.

### 4. Build your image

```bash
$ bitbake your-image
```

The agent is included in the image and starts automatically on boot.

## Accept the device

After booting the image, the device appears as **Pending** on the Devices page in the dashboard. Accept it to start connecting.
