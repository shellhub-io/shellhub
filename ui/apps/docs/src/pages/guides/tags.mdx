# Tags

Tags are labels you assign to devices to organize your fleet. Use them to group by environment, location, role, or anything that fits your workflow.

## Assigning tags

On the **Devices** page, open a device's **Actions** menu and click **Edit tags**. Enter tags separated by commas or spaces (for example, `production, web`), then save. Each device supports up to 3 tags, and tag names can contain lowercase letters, numbers, and hyphens.

## Filtering

On the **Devices** page, use the **Tags** menu to filter the list by one or more tags. Combine tags for more specific views.

## Managing tags globally

Manage every tag in your namespace from one place: go to **Settings → Tags**, where you can rename or remove any tag across all devices.

## Tags + access control

Tags drive two of ShellHub's access-control features:

- **[Public keys](/guides/public-keys)** — restrict a public key to devices with specific tags, so the key only authenticates against those devices.
- **[Firewall rules](/guides/firewall-rules)** — scope a rule to tagged devices, applying allow/deny policies to a group instead of individual hosts.

For example:

- Tag production servers with `production`, staging with `staging`
- Create a deploy key filtered to `production` — it only works on production devices
- Create a dev key filtered to `staging` — it only works on staging devices

This gives you environment-level access control without managing individual device permissions.

## Common patterns

| Tags | Purpose |
|------|---------|
| `production`, `staging`, `dev` | Environment separation |
| `us-east`, `eu-west` | Geographic location |
| `web`, `database`, `worker` | Server role |

## Editing tags

Tags can be added or removed at any time without affecting device connectivity. Removing a tag immediately updates any public key or firewall rule filters that reference it.
